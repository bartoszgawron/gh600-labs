# Security Policy

## Supported Versions

Security updates are provided on a best-effort basis for the latest state of the default branch (`main`).

## Reporting a Vulnerability

If you discover a security vulnerability, please report it privately.

- **Do not** open a public issue for security problems.
- Use GitHub's private vulnerability reporting flow: go to the repository **Security** tab and choose **Report a vulnerability**.
- Include clear reproduction steps, impact, and any suggested remediation.

## What to Expect

- We will acknowledge receipt as soon as possible.
- We will investigate and validate the report.
- Once confirmed, we will prepare and publish a fix.

Thank you for helping keep this repository and its users safe.
